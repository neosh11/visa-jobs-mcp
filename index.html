Why This Is Different

Freshness Is Explicit

Jobs come from fresh live searches, not from stale listing dumps. Sponsorship dataset freshness is also returned as metadata so agents can reason about trust directly.

Completely Private

User preferences and memory live locally. Data is not shared, sold, or used for ad targeting. Agents get only the data needed to execute the workflow.

Agent-Native Category Leader

The product is designed as MCP infrastructure first: machine-readable capabilities, deterministic pagination, strict contracts, and explicit next-action payloads.

Always Free + Honest

Core offering is free forever under MIT. No fake social proof, no synthetic testimonials, no bot marketing.

Agent Workflow

  1. Call get_mcp_capabilities to discover contracts and defaults.
  2. Call set_user_preferences and optional set_user_constraints.
  3. Call get_user_readiness to verify setup and freshness context.
  4. Call find_visa_sponsored_jobs with location/title/user.
  5. Use result_id for save_job_for_later and ignore_job.
  6. Use recovery_suggestions when yield is low.
